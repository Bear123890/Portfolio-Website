## Overview

This is a personal portfolio website designed to showcase my background, skills, and experience to potential employers and collaborators. The entire site lives in a single `index.html` file and is hosted for free using **GitHub Pages**. No frameworks, no backend — just HTML, CSS, and JavaScript.

## How the Site Was Built

This site was built using **Claude AI (claude.ai)** as the primary development tool. Here is the step-by-step process:

### Step 1 — Generate the Website with Claude AI
I opened [claude.ai](https://claude.ai) and prompted Claude to build a complete portfolio website based on my assignment requirements. Claude generated all of the HTML, CSS, and JavaScript needed for the site in a single code block.

### Step 2 — Fill in Content with My Resume
After the site structure was generated, I uploaded my resume to Claude and asked it to populate the website content (About section, Skills, Projects, etc.) using the information from my resume. Claude replaced all placeholder text with my real background and experience.

### Step 3 — Fix the QR Code
The generated site included a QR code, but it was pointing to an incorrect or placeholder URL. I manually went into the `index.html` code and updated the QR code so it correctly linked to my resume document.

### Step 4 — Encode the Resume as Base64 for Download
To make the resume downloadable directly from the page (without needing to host a separate PDF file), I converted my resume PDF into a **Base64 string**. This encodes the entire PDF as text that can be embedded right inside the HTML file. When a visitor clicks "Download Resume," the browser decodes it and saves the file locally. No external file hosting is needed.

### Step 5 — Deploy to GitHub Pages
1. Created a new repository on [github.com](https://github.com)
2. Uploaded the `index.html` file to the repository
3. Went to **Settings → Pages** and enabled GitHub Pages on the `main` branch
4. The site went live at `https://bear123890.github.io/Portfolio-Website/`

## How the Chatbot Was Implemented

The chatbot on this site was generated by Claude AI as part of the initial website build. It is built entirely with **vanilla JavaScript** embedded in the `index.html` file — no external libraries or backend server required.

### How It Works

1. A chat icon is displayed on the page (typically in the bottom corner or in its own section).
2. The visitor types a question and submits it.
3. The JavaScript reads the input and matches it against a set of **predefined responses** about my background, skills, and experience.
4. The matched response is displayed in the chat window.

All chatbot logic and responses are written directly into the `index.html` file by Claude AI during the generation step, using the resume content I provided to personalize the answers.

## How the Downloadable Resume Works

The resume download feature uses **Base64 encoding** to embed the PDF directly inside the HTML file.

### What Is Base64?

Base64 is a way of converting binary files (like a PDF) into a long string of plain text characters. That text string can be stored inside HTML and decoded back into the original file by the browser when needed.

### How It's Done

1. Convert your resume PDF to Base64 using an online tool (search "PDF to Base64 converter").
2. Copy the resulting Base64 string.
3. In the `index.html` file, the download button's `href` is set like this:

```html
<a href="data:application/pdf;base64,[YOUR_BASE64_STRING_HERE]"
   download="Resume.pdf">
  Download Resume
</a>
```

4. When a visitor clicks the button, the browser decodes the Base64 string back into a PDF and saves it as `Resume.pdf`.

**Benefit:** The entire resume is self-contained inside `index.html` — no separate file upload to GitHub needed.

## How the QR Code Works

The site includes a QR code that visitors can scan with their phone to access the resume directly.

The QR code was generated to point to a specific URL. After Claude AI produced the initial version, the target URL was **manually corrected** in the HTML to point to the right document. If you ever need to regenerate the QR code:

1. Use a free generator like [qr-code-generator.com](https://www.qr-code-generator.com) pointing to your desired URL.
2. Replace the QR code image source in the `index.html` file.
